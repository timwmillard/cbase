## Arena API

ecewo has its own arena allocator, which is a customized version of [tsoding/arena](https://github.com/tsoding/arena). Here are the functions available:

```c
ecewo_arena_t *ecewo_arena_borrow(void);
void ecewo_arena_return(ecewo_arena_t *arena);
void *ecewo_alloc(ecewo_arena_t *arena, size_t size_bytes);
void *ecewo_realloc(ecewo_arena_t *arena, void *oldptr, size_t oldsz, size_t newsz);
char *ecewo_strdup(ecewo_arena_t *arena, const char *cstr);
char *ecewo_sprintf(ecewo_arena_t *arena, const char *format, ...);
void *ecewo_memdup(ecewo_arena_t *arena, void *data, size_t size);
```

## Usage

Every handler in ecewo has an arena. You can access them by `req->arena` or `res->arena`. Both of them point the same arena, so you can use either one.

The handler's arena is going to be freed when you send a response. But if you want to create a separate arena for reason (for fire-and-forget tasks for example), you are responsible to free it.

### Using Handler's Arena

```c
#include "ecewo.h"

void arena_example(ecewo_request_t *req, ecewo_response_t *res) {
  // Allocate memory (automatically freed after response)
  char *buffer = ecewo_alloc(req->arena, 1024);
  
  // Duplicate string
  char *copy = ecewo_strdup(req->arena, "original string");
  
  // Formatted string allocation
  char *formatted = ecewo_sprintf(req->arena, "User %d has %s access", 
                   123, "admin");
  
  // Memory duplication
  char data[] = "binary data";
  void *dup = ecewo_memdup(req->arena, data, sizeof(data));
  
  // Reallocation
  char *new_buffer = ecewo_realloc(req->arena, buffer, 1024, 2048);
  
  ecewo_send_text(res, 200, formatted); // Send a response, arena will be freed automatically
}
```

### Using Custom Arena

```c
#include "ecewo.h"
#include <stdio.h>

void custom_arena_example(void) {
  // Borrow an arena from the pool
  ecewo_arena_t *custom_arena = ecewo_arena_borrow();
  
  // Allocate from custom arena
  char *data1 = ecewo_strdup(custom_arena, "Hello");
  char *data2 = ecewo_sprintf(custom_arena, "Number: %d", 42);
  int *numbers = ecewo_alloc(custom_arena, sizeof(int) * 100);
  
  // Use the data
  char *response = ecewo_sprintf(custom_arena, "%s - %s", data1, data2);
  
  printf("%s\n", response);
  
  // Give it back to the pool
  ecewo_arena_return(custom_arena);
}
```

### Using Handler's Arena Out of The Handler

You may want to use the existing arena that handler provides in a custom function (see the [Workers Chapter](07.workers.md)). That's totally fine, but you must be careful while doing that. The handler's arena is designed to be freed automatically when a response has been sent. So if you want to use the handler's arena outside of the handler, you have to send a response to the client.

If you want to use the handler's arena outside of the handler, **DO NOT DO THIS:**

```c
typedef struct
{
  ecewo_arena_t *arena;
  int result;
} Context;

void handler(ecewo_request_t *req, ecewo_response_t *res) {
  // Create context
  Context *ctx = ecewo_alloc(res->arena, sizeof(Context));
  ctx->arena = res->arena;
  ctx->result = 0;
  
  // Imagine a function call here that will use the Context
}
```

This will break your application, because you'll never be able to send a response. To send a response and free the handler's arena, pass the Res struct's itself to the context, not the Arena.

**DO THIS INSTEAD:**

```c
typedef struct
{
  ecewo_response_t *res;
  int result;
} Context;

void handler(ecewo_request_t *req, ecewo_response_t *res) {
  // Create context
  Context *ctx = ecewo_alloc(res->arena, sizeof(Context));
  ctx->res = res;
  ctx->result = 0;
  
  // Imagine a function call here that will use the Context
}
```

See the [Workers Chapter](07.workers.md) for more advanced examples.

> [!NOTE]
> 
> It's totally fine to use dynamic memory allocation functions that `stdlib.h` provides, but it's strongly recommended to use the arena allocator that ecewo offers.

## Cleanup App Resources

ecewo provides `ecewo_atexit()` for one last cleanup right before shutting down the server. We can use it for such cases like saving critical data, completing pending transactions, closing database connections, final logs, cleaning up caches etc.

```c
// main.c

#include "ecewo.h"
#include <stdio.h>

// User resources cleanup
void app_cleanup(void *user_data) {
  (void)user_data;
  db_close();
  redis_close();

  printf("App cleaned up\n");
}

int main(void) {
  ecewo_app_t *app = ecewo_create();
  if (!app) {
    fprintf(stderr, "Failed to initialize server\n");
    return 1;
  }

  db_init();
  redis_init();

  // Register app_cleanup function
  // It's going to work right before shutdown
  if (ecewo_atexit(app, app_cleanup, NULL) != 0) {
    fprintf(stderr, "Failed to register shutdown callback\n");
    return 1;
  }

  if (ecewo_listen(app, 3000) != 0) {
    fprintf(stderr, "Failed to start server\n");
    return 1;
  }

  return 0;
}
```

> [!IMPORTANT]
>
> Need to register `ecewo_atexit()` right before `ecewo_listen()`.

## Monitoring Arena Usage

Enable debug mode to see arena statistics:

```shell
target_compile_definitions(my_app PRIVATE ECEWO_DEBUG=1)
```

Check arena pool status in your application:
```c
#ifdef ECEWO_DEBUG
ecewo_arena_pool_stats();
#endif
```

Output example:

```shell
Arena Pool Statistics:
  Available: 45/128 arenas (2.81 MB)
  In use: 83 arenas
  Peak usage: 95 arenas
  Total allocated: 8.00 MB
  Grow operations: 3
  Shrink operations: 1
```
