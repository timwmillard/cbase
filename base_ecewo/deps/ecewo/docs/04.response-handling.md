## Response Functions

### `ecewo_send()`

`ecewo_send()` is the main response function. We can call it to send any type of response.

```c
void ecewo_send(ecewo_response_t *res, int status, const void *body, size_t body_len);
```

> [!NOTE]
>
> `ecewo_send()` does not set a `Content-Type` header, which is necessary for sending a response. You must call `ecewo_header_set()` before calling `ecewo_send()`. See the examples below.

**Usage:**

```c
#include "ecewo.h"

void hello_world(ecewo_request_t *req, ecewo_response_t *res) {
  ecewo_header_set(res, "Content-Type", "text/plain");
  ecewo_send(res, 200, "Hello, World!", strlen("Hello, World!"));
}
```

We can also write our own abstractions over it for content types we frequently use. For example:

```c
static void ecewo_send_cbor(ecewo_response_t *res, int status, const void *body, size_t body_len) {
  ecewo_header_set(res, "Content-Type", "application/cbor");
  ecewo_send(res, status, body, body_len);
}

void handler(ecewo_request_t *req, ecewo_response_t *res) {
  // Some CBOR things...
  ecewo_send_cbor(res, 200, body, body_len);
}
```

That said, ecewo already provides some abstractions for frequently using responses, such as [ecewo_send_text()](#ecewo_send_text), [ecewo_send_json()](#ecewo_send_json) and [ecewo_send_html()](#ecewo_send_html).

### `ecewo_send_text()`

```c
#include "ecewo.h"

void hello_world(ecewo_request_t *req, ecewo_response_t *res) {
   ecewo_send_text(res, 200, "Hello, World!");
}
```

### `ecewo_send_json()`

We can easily generate and parse JSON objects via [cJSON](https://github.com/DaveGamble/cJSON) or [jansson](https://github.com/akheron/jansson). See [Using JSON](/docs/examples/using_json.md) chapter for more detailed usage.

```c
#include "ecewo.h"

void hello_world(ecewo_request_t *req, ecewo_response_t *res) {
  ecewo_send_json(res, 200, "{ \"message\": \"Hello, World!\" }");
}
```

### `ecewo_send_html()`

```c
#include "ecewo.h"

void home_handler(ecewo_request_t *req, ecewo_response_t *res) {
  const char *html = 
    "<!DOCTYPE html>"
    "<html>"
    "<head>"
    "  <title>Welcome</title>"
    "</head>"
    "<body>"
    "  <h1>Hello from ecewo!</h1>"
    "  <p>This is a simple HTML page.</p>"
    "</body>"
    "</html>";
  
  ecewo_send_html(res, 200, html);
}
```

## Redirecting

```c
void moved_permanently_handler(ecewo_request_t *req, ecewo_response_t *res) {
  ecewo_redirect(res, 301, "https://newdomain.com/page");
}

void found_handler(ecewo_request_t *req, ecewo_response_t *res) {
  ecewo_redirect(res, 302, "/login");
}
```

See the [3xx](#status-code-enums) codes if you would like to use `enums` instead of status codes.

```c
  // 3xx Redirection
  ECEWO_MULTIPLE_CHOICES = 300,

  ECEWO_USE_PROXY = 305,

  ECEWO_PERMANENT_REDIRECT = 308,

  ECEWO_FORBIDDEN = 403,
  ECEWO_NOT_FOUND = 404,

  ECEWO_URI_TOO_LONG = 414,
  ECEWO_UNSUPPORTED_MEDIA_TYPE = 415,

  ECEWO_NOT_EXTENDED = 510,
  ECEWO_NETWORK_AUTHENTICATION_REQUIRED = 511

```

Example response sending with status codes:

```c
ecewo_send_text(res, 200, "Success");
ecewo_send_text(res, 201, "Content created")
ecewo_send_text(res, 400, "Bad request");
ecewo_send_text(res, 404, "Not found");
ecewo_send_text(res, 500, "Internal server error");
```

Example response sending with status code enums:

```c
ecewo_send_text(res, ECEWO_OK, "Success");
ecewo_send_text(res, ECEWO_CREATED, "Content created")
ecewo_send_text(res, ECEWO_BAD_REQUEST, "Bad request");
ecewo_send_text(res, ECEWO_NOT_FOUND, "Not found");
ecewo_send_text(res, ECEWO_INTERNAL_SERVER_ERROR, "Internal server error");
```

They all will work.

## Custom Headers

```c
void custom_headers(ecewo_request_t *req, ecewo_response_t *res) {
  ecewo_header_set(res, "X-API-Version", "1.0");
  ecewo_header_set(res, "Cache-Control", "no-cache");
  ecewo_header_set(res, "X-Rate-Limit", "100");
  
  ecewo_send_text(res, 200, "Success!");
}
```

> [!WARNING]
>
> `ecewo_header_set()` does NOT dedupe arbitrary headers; calling it twice with the same
> name appends both (this is legitimate for `Set-Cookie`).
>
> The following framing / hop-by-hop headers are reserved by the framework and rejected
> if you try to set them: `Content-Length`, `Transfer-Encoding`, `Connection`, `Host`,
> `Date`. Attempts are logged at ERROR and ignored.
>
> DO NOT set `Content-Type` header if you use:
> - `ecewo_send_text()`
> - `ecewo_send_json()`
> - `ecewo_send_html()`
> - `ecewo_redirect()`

**Incorrect usage:**
```c
ecewo_header_set(res, "Content-Type", "text/xml");
ecewo_send_json(res, 200, json);
```

**Correct usage - Option 1: Use helper only**
```c
ecewo_send_json(res, 200, json);
```

**Correct usage - Option 2: Manual control**
```c
ecewo_header_set(res, "Content-Type", "application/json");
ecewo_header_set(res, "Cache-Control", "no-cache");
ecewo_send(res, 200, json, json_len);
```

**Correct usage - Option 3: Multiple values (Set-Cookie)**
```c
ecewo_header_set(res, "Set-Cookie", "session=abc; HttpOnly");
ecewo_header_set(res, "Set-Cookie", "token=xyz; Secure");
```
