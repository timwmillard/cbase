# String Builder

**String builder** is a dynamic array of `char` with helpers for appending C strings and null-terminating the result.

## API

### StringBuilder convention

ecewo does not provide a built-in `StringBuilder` type. The `ECEWO_SB_*` macros work on any struct that has `items`, `count`, and `capacity` fields (the same layout the dynamic array macros expect). Define the struct yourself:

```c
typedef struct {
  char *items;     // character buffer (not null-terminated until ECEWO_SB_APPEND_NULL)
  size_t count;    // number of characters written
  size_t capacity; // allocated capacity in bytes
} StringBuilder;
```

Because it follows the dynamic array convention, `ECEWO_DA_APPEND` and `ECEWO_DA_APPEND_MANY` work on it directly; the `ECEWO_SB_*` macros are thin wrappers around those.

Initialize to all-zeros before use:

```c
StringBuilder sb = {0};
```

### `ECEWO_SB_APPEND_CSTR`

```c
ECEWO_SB_APPEND_CSTR(ecewo_arena_t *arena, StringBuilder *sb, const char *cstr);
```

Appends the null-terminated string `cstr` to `sb`, excluding the terminator. Internally calls `strlen` and then `ECEWO_DA_APPEND_MANY`.

Use this when appending string literals, `ecewo_sprintf` results, or any other null-terminated string.

**Example**

```c
StringBuilder sb = {0};
ECEWO_SB_APPEND_CSTR(req->arena, &sb, "Hello, ");
ECEWO_SB_APPEND_CSTR(req->arena, &sb, "world");
ECEWO_SB_APPEND_NULL(req->arena, &sb);

// sb.items = "Hello, world"
```

To append a buffer of known length without a `strlen` call; for example a string literal whose size is statically known; use `ECEWO_DA_APPEND_MANY` directly:

```c
static const char header[] = "id,name,email\n";
ECEWO_DA_APPEND_MANY(req->arena, &sb, header, sizeof(header) - 1); // -1 to exclude '\0'
```

### ECEWO_SB_APPEND_NULL

```c
ECEWO_SB_APPEND_NULL(ecewo_arena_t *a, StringBuilder *sb);
```

Appends a single `'\0'` byte to `sb`. Must be called before using `sb.items` as a C string.

> [!IMPORTANT]
>
> `sb.items` is **not** null-terminated until you call `ECEWO_SB_APPEND_NULL`. Passing `sb.items` to any function that expects a C string before this call is undefined behavior.

When passing the result to `ecewo_send()` directly, exclude the null terminator from the length:

```c
ECEWO_SB_APPEND_NULL(req->arena, &sb);
ecewo_send(res, ECEWO_OK, sb.items, sb.count - 1);
```

When passing to `ecewo_send_json()` or `ecewo_send_html()`, pass `sb.items` as-is after null-terminating:

```c
ECEWO_SB_APPEND_NULL(req->arena, &sb);
ecewo_send_json(res, ECEWO_OK, sb.items);
```

## Examples

### CSV export

A `GET /export/orders` endpoint that streams all orders as a CSV file. The column header is a string literal with a statically known size, so `ECEWO_DA_APPEND_MANY` with `sizeof - 1` avoids an unnecessary `strlen`. Each data row is formatted with `ecewo_sprintf` and appended with `ECEWO_SB_APPEND_CSTR`.

```c
#include "ecewo.h"

void export_orders_csv(ecewo_request_t *req, ecewo_response_t *res) {
  DbResult *rows = db_query("SELECT id, customer, total, status FROM orders");

  static const char header[] = "id,customer,total,status\n";

  StringBuilder sb = {0};

  // sizeof - 1 excludes the null terminator; no strlen needed
  ECEWO_DA_APPEND_MANY(req->arena, &sb, header, sizeof(header) - 1);

  for (int i = 0; i < rows->count; i++) {
    char *line = ecewo_sprintf(req->arena, "%d,%s,%.2f,%s\n",
      rows->data[i].id,
      rows->data[i].customer,
      rows->data[i].total,
      rows->data[i].status);
    ECEWO_SB_APPEND_CSTR(req->arena, &sb, line);
  }

  ECEWO_SB_APPEND_NULL(req->arena, &sb);

  ecewo_header_set(res, "Content-Type", "text/csv");
  ecewo_header_set(res, "Content-Disposition", "attachment; filename=\"orders.csv\"");
  ecewo_send(res, ECEWO_OK, sb.items, sb.count - 1);
}
```

### Plain text report

A `GET /status` endpoint that returns a human-readable summary as `text/plain`. The response has a fixed header section followed by one line per service, so the final size is unknown upfront.

```c
#include "ecewo.h"

typedef struct {
  const char *name;
  bool up;
  int latency_ms;
} ServiceStatus;

void get_status(ecewo_request_t *req, ecewo_response_t *res) {
  ServiceStatus services[] = {
    { "database", true, 4  },
    { "cache", true, 1 },
    { "email", false, 0 },
    { "file-store", true, 12 },
  };
  int count = 4;

  StringBuilder sb = {0};

  ECEWO_SB_APPEND_CSTR(req->arena, &sb, "SERVICE STATUS REPORT\n");
  ECEWO_SB_APPEND_CSTR(req->arena, &sb, "=====================\n\n");

  for (int i = 0; i < count; i++) {
    char *line = ecewo_sprintf(req->arena, "%-16s %s%s\n",
      services[i].name,
      services[i].up ? "UP" : "DOWN",
      services[i].up
        ? ecewo_sprintf(req->arena, "  (%d ms)", services[i].latency_ms)
        : "");
    ECEWO_SB_APPEND_CSTR(req->arena, &sb, line);
  }

  ECEWO_SB_APPEND_NULL(req->arena, &sb);

  ecewo_header_set(res, "Content-Type", "text/plain");
  ecewo_send(res, ECEWO_OK, sb.items, sb.count - 1);
}
```

Output:

```
SERVICE STATUS REPORT
=====================

database         UP  (4 ms)
cache            UP  (1 ms)
email            DOWN
file-store       UP  (12 ms)
```

### Dynamic redirect URL

A login handler that redirects to the originally requested URL after authentication, including any extra query parameters that were passed in. The final redirect target is built piece by piece.

```c
#include "ecewo.h"

void post_login(ecewo_request_t *req, ecewo_response_t *res) {
  const char *username = ecewo_query(req, "username");
  const char *password = ecewo_query(req, "password");
  const char *next = ecewo_query(req, "next");
  const char *ref = ecewo_query(req, "ref");

  if (!authenticate(username, password)) {
    ecewo_send_text(res, ECEWO_UNAUTHORIZED, "Invalid credentials");
    return;
  }

  // Build the redirect target
  StringBuilder sb = {0};

  // Base path: caller-supplied or default
  ECEWO_SB_APPEND_CSTR(req->arena, &sb, next ? next : "/dashboard");

  // Append optional tracking reference as a query parameter
  if (ref) {
    ECEWO_SB_APPEND_CSTR(req->arena, &sb, "?ref=");
    ECEWO_SB_APPEND_CSTR(req->arena, &sb, ref);
  }

  ECEWO_SB_APPEND_NULL(req->arena, &sb);

  ecewo_redirect(res, FOUND, sb.items);
}
```

### Dynamic Allow header

A `405 Method Not Allowed` helper that reads which methods are registered for a route and builds the `Allow` header value as a comma-separated list.

```c
#include "ecewo.h"

void method_not_allowed(ecewo_request_t *req, ecewo_response_t *res, const char **methods, int count) {
  StringBuilder sb = {0};

  for (int i = 0; i < count; i++) {
    if (i > 0) ECEWO_SB_APPEND_CSTR(req->arena, &sb, ", ");
    ECEWO_SB_APPEND_CSTR(req->arena, &sb, methods[i]);
  }

  ECEWO_SB_APPEND_NULL(req->arena, &sb);

  ecewo_header_set(res, "Allow", sb.items);
  ecewo_send_text(res, ECEWO_METHOD_NOT_ALLOWED, "Method Not Allowed");
}

// Usage:
//   const char *allowed[] = {"GET", "POST", "OPTIONS"};
//   method_not_allowed(req, res, allowed, 3);
//
// Produces:  Allow: GET, POST, OPTIONS
```
