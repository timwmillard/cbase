Linkage: every entry below is `extern "C"` and exported from the shared library on platforms where visibility applies.

## Types

### Opaque handles

All public types are forward-declared structs. The struct definitions are private - never dereference these pointers, only pass them to `ecewo_*` functions.

| Type                          | Purpose                                                              |
| ----------------------------- | -------------------------------------------------------------------- |
| `ecewo_app_t *`               | Application instance. Created by `ecewo_create()`.                   |
| `ecewo_route_t *`             | Route builder. Created by `ecewo_route_new()`, finalized by `ecewo_route_handler()`. |
| `ecewo_request_t *`           | Incoming HTTP request. Passed to handlers and middleware.            |
| `ecewo_response_t *`          | Outgoing HTTP response. Passed to handlers and middleware.           |
| `ecewo_client_t *`            | Underlying TCP client. For plugin authors only.                      |
| `ecewo_timer_t *`             | Timer handle returned by `ecewo_timeout()` / `ecewo_interval()`.     |
| `ecewo_arena_t *`             | Arena allocator.                                                     |
| `ecewo_takeover_config_t *`   | Connection-takeover configuration.                                   |

### Function pointer types

```c
typedef void (*ecewo_next_t)      (ecewo_request_t *, ecewo_response_t *);
typedef void (*ecewo_handler_t)   (ecewo_request_t *req, ecewo_response_t *res);

typedef void (*ecewo_body_data_cb_t)(ecewo_request_t *req, const uint8_t *data, size_t len);

```

Connection-takeover callbacks follow libuv signatures (`uv_alloc_cb`, `uv_read_cb`, `uv_close_cb`) and are passed as `void *`.

## App lifecycle

### `ecewo_create`

```c
ecewo_app_t *ecewo_create(void);
```

Allocate and initialize a new application instance with default configuration. Returns `NULL` on allocation failure. The first call lazily initializes the process-level runtime (event loop, signal handlers); subsequent calls attach the new app to the same runtime.

### `ecewo_bind`

```c
int ecewo_bind(ecewo_app_t *app, uint16_t port);
```

Bind the server to `port` without entering the event loop. Use this when you need to register timers or perform other setup before calling `ecewo_run()`, or to bind multiple apps before starting the shared loop. Returns `0` on success, `-1` on error.

### `ecewo_run`

```c
void ecewo_run(void);
```

Start the shared event loop. Blocks until every registered app has been shut down (or a signal stops the process). Recursive calls are no-ops.

### `ecewo_listen`

```c
int ecewo_listen(ecewo_app_t *app, uint16_t port);
```

Bind to `port` and start the event loop in a single call. Equivalent to `ecewo_bind()` + `ecewo_run()`. Blocks until shutdown. Returns 0 on success, -1 on error.

### `ecewo_shutdown`

```c
void ecewo_shutdown(ecewo_app_t *app);
```

Initiate a graceful shutdown of this app. Closes the listener, drains active connections up to `shutdown_timeout_ms`, then forcibly closes them, fires the app's `ecewo_atexit` callback, and unregisters the app from the runtime. Safe to call from inside a request handler. In a multi-app process, shutting down one app does not stop the others.

### `ecewo_atexit`

```c
int ecewo_atexit(ecewo_app_t *app,
                 void (*callback)(void *user_data),
                 void *user_data);
```

Register a callback to be invoked during shutdown, before the event loop exits. Use for releasing resources such as database connections or thread pools. `user_data` is passed to `callback` unchanged. Returns 0 on success, -1 on error.

## Timers

### `ecewo_timeout`

```c
ecewo_timer_t *ecewo_timeout(ecewo_timer_cb_t callback, uint64_t delay_ms, void *user_data);
```

Schedule `callback` to be called once after `delay_ms` milliseconds. Returns an opaque timer handle, or `NULL` on error. The timer is automatically freed after it fires.

### `ecewo_interval`

```c
ecewo_timer_t *ecewo_interval(ecewo_timer_cb_t callback, uint64_t interval_ms, void *user_data);
```

Schedule `callback` to be called repeatedly every `interval_ms` milliseconds. Returns an opaque timer handle, or `NULL` on error. Call `ecewo_clear_timer()` to stop it.

### `ecewo_clear_timer`

```c
void ecewo_clear_timer(ecewo_timer_t *timer);
```

Cancel and free a timer returned by `ecewo_timeout()` or `ecewo_interval()`. After this call the handle is invalid.

### `ecewo_timeout_request`

```c
int ecewo_timeout_request(ecewo_response_t *res, uint64_t timeout_ms);
```

Enable a per-request deadline. If the handler does not send a response within `timeout_ms`, the connection is closed. Calling again resets the timeout. Returns `0` on success, `-1` on error.

## Request accessors

All accessors are read-only. Returned pointers are arena-owned and valid until the response is sent.

### `ecewo_param`

```c
const char *ecewo_param(const ecewo_request_t *req, const char *key);
```

Return the value of a named path parameter, or `NULL` if absent. For a route `"/users/:id"`, `ecewo_param(req, "id")` returns the captured segment.

### `ecewo_query`

```c
const char *ecewo_query(const ecewo_request_t *req, const char *key);
```

Return the value of a named query string parameter, or `NULL` if absent. For `/search?q=hello`, `ecewo_query(req, "q")` returns `"hello"`.

### `ecewo_header_get`

```c
const char *ecewo_header_get(const ecewo_request_t *req, const char *key);
```

Return the value of an incoming request header (case-insensitive), or `NULL` if absent.

### `ecewo_req_app`

```c
ecewo_app_t *ecewo_req_app(const ecewo_request_t *req);
```

Return the application instance associated with the request.

### `ecewo_req_arena`

```c
ecewo_arena_t *ecewo_req_arena(const ecewo_request_t *req);
```

Return the per-request arena allocator. All allocations live until the response is sent.

### `ecewo_req_method`

```c
const char *ecewo_req_method(const ecewo_request_t *req);
```

Return the HTTP method string (`"GET"`, `"POST"`, ...).

### `ecewo_req_path`

```c
const char *ecewo_req_path(const ecewo_request_t *req);
```

Return the request path (e.g. `"/users/42"`).

### `ecewo_req_body`

```c
const uint8_t *ecewo_req_body(const ecewo_request_t *req);
```

Return the raw request body bytes, or `NULL` when body streaming is active. **Not null-terminated** - pair with `ecewo_req_body_len()`.

### `ecewo_req_body_len`

```c
size_t ecewo_req_body_len(const ecewo_request_t *req);
```

Return the number of bytes in the request body.

### `ecewo_req_http_major`

```c
uint8_t ecewo_req_http_major(const ecewo_request_t *req);
```

HTTP major version (`1` for HTTP/1.x).

### `ecewo_req_http_minor`

```c
uint8_t ecewo_req_http_minor(const ecewo_request_t *req);
```

HTTP minor version (`0` or `1`).

### `ecewo_req_is_head`

```c
bool ecewo_req_is_head(const ecewo_request_t *req);
```

`true` when the request method is `HEAD`. Response body is automatically suppressed by the framework on HEAD requests.

### `ecewo_req_client`

```c
ecewo_client_t *ecewo_req_client(ecewo_request_t *req);
```

Return the client handle associated with a request. For plugin authors only.

## Response

After any of the response-sending functions returns, the `ecewo_response_t *` handle must not be accessed again - the request has been finalized.

### `ecewo_header_set`

```c
void ecewo_header_set(ecewo_response_t *res, const char *name, const char *value);
```

Append a response header. Calling twice with the same name produces two headers (legitimate for `Set-Cookie`). Reserved framing / hop-by-hop headers are silently rejected: `Content-Length`, `Transfer-Encoding`, `Connection`, `Host`, `Date`.

### `ecewo_send`

```c
void ecewo_send(ecewo_response_t *res, int status, const void *body, size_t body_len);
```

Send a response with the given status code and raw body. `body` may be `NULL` when `body_len == 0` (e.g. for `204 No Content`). The body buffer is copied; the caller does not need to keep it alive.

### `ecewo_redirect`

```c
void ecewo_redirect(ecewo_response_t *res, int status, const char *url);
```

Send a 3xx redirect with the `Location` header set to `url`.

### `ecewo_send_text`

```c
void ecewo_send_text(ecewo_response_t *res, int status, const char *body);
```

Send a plain-text response (`Content-Type: text/plain; charset=utf-8`).

### `ecewo_send_html`

```c
void ecewo_send_html(ecewo_response_t *res, int status, const char *body);
```

Send an HTML response (`Content-Type: text/html; charset=utf-8`).

### `ecewo_send_json`

```c
void ecewo_send_json(ecewo_response_t *res, int status, const char *body);
```

Send a JSON response (`Content-Type: application/json`).

### `ecewo_res_arena`

```c
ecewo_arena_t *ecewo_res_arena(const ecewo_response_t *res);
```

Return the per-request arena allocator shared with the request.

### `ecewo_res_client`

```c
ecewo_client_t *ecewo_res_client(ecewo_response_t *res);
```

Return the client handle associated with a response. For plugin authors only.

## Memory management

ecewo allocates through arenas. See `06.memory-management.md` for the model and `07.workers.md` for arena rules across thread boundaries.

### `ecewo_alloc`

```c
void *ecewo_alloc(ecewo_arena_t *arena, size_t size_bytes);
```

Allocate `size_bytes` from `arena`. Returns `NULL` on failure. Aligned for any standard type.

### `ecewo_realloc`

```c
void *ecewo_realloc(ecewo_arena_t *arena, void *oldptr, size_t oldsz, size_t newsz);
```

Grow or shrink an allocation. The caller must pass the previous size as `oldsz`. Returns the new pointer, or `NULL` on failure.

### `ecewo_strdup`

```c
char *ecewo_strdup(ecewo_arena_t *arena, const char *cstr);
```

Allocate and copy a null-terminated string into the arena.

### `ecewo_memdup`

```c
void *ecewo_memdup(ecewo_arena_t *arena, void *data, size_t size);
```

Allocate and copy `size` bytes into the arena.

### `ecewo_sprintf`

```c
char *ecewo_sprintf(ecewo_arena_t *arena, const char *format, ...);
```

`printf`-into-arena. Returns the allocated, null-terminated result.

### `ecewo_arena_borrow`

```c
ecewo_arena_t *ecewo_arena_borrow(void);
```

Take an arena from the pool for short-lived scratch work. Must be paired with `ecewo_arena_return()`.

### `ecewo_arena_return`

```c
void ecewo_arena_return(ecewo_arena_t *arena);
```

Return an arena previously taken with `ecewo_arena_borrow()`. The arena's contents are reset and the arena returns to the pool.

### `ecewo_arena_pool_stats`

```c
void ecewo_arena_pool_stats(void);
```

Print pool statistics to stderr. Only available when ecewo was built with `-DCMAKE_BUILD_TYPE=Debug` (i.e. `ECEWO_DEBUG` defined).

## App configuration

All setters must be called *before* `ecewo_bind()` / `ecewo_listen()`. See `10.configurations.md` for guidance on tuning.

| Function                              | Default | Description                                           |
| ------------------------------------- | ------- | ----------------------------------------------------- |
| `ecewo_set_max_connections(app, n)`   | 10000   | Maximum simultaneous TCP connections.                 |
| `ecewo_set_listen_backlog(app, n)`    | 511     | TCP `listen(2)` backlog.                              |
| `ecewo_set_idle_timeout(app, ms)`     | 60000   | Idle connection timeout in ms; `0` disables.          |
| `ecewo_set_request_timeout(app, ms)`  | 0       | Per-request timeout in ms; `0` disables.              |
| `ecewo_set_cleanup_interval(app, ms)` | 30000   | How often the cleanup timer runs.                     |
| `ecewo_set_shutdown_timeout(app, ms)` | 15000   | Graceful shutdown drain timeout.                      |
| `ecewo_set_listen_address(app, addr)` | "0.0.0.0" | Numeric IPv4/IPv6 bind address. No hostname lookup. |

### `ecewo_set_listen_address`

```c
void ecewo_set_listen_address(ecewo_app_t *app, const char *address);
```

Examples: `"127.0.0.1"`, `"::"` (all IPv6; on dual-stack systems this also accepts IPv4), `"::1"`, `"192.168.1.10"`.

## Middleware registration

### `ecewo_use`

```c
int ecewo_use(ecewo_app_t *app, const char *path, ecewo_middleware_t fn);
```

Register a middleware function. If `path` is `NULL`, `fn` runs for every request. If non-`NULL`, `fn` runs only when the request path starts with that prefix. Middlewares execute in registration order before the route handler; `fn` must call `next(req, res)` to continue or send a response to short-circuit. Returns `0` on success, `-1` on error.

See `05.middleware.md` for examples.

## Route registration

The builder API (`ecewo_route_new` / `ecewo_route_middleware` / `ecewo_route_handler`) is the recommended way to register routes from non-C code or from data tables. C application code typically uses the convenience macros (`ECEWO_GET`, etc.).

### `ecewo_route_new`

```c
ecewo_route_t *ecewo_route_new(ecewo_app_t *app, ecewo_method_t method, const char *path);
```

Start building a route. Returns an opaque builder handle, or `NULL` on error. The builder is arena-allocated and does not need to be freed.

### `ecewo_route_middleware`

```c
void ecewo_route_middleware(ecewo_route_t *route, ecewo_middleware_t fn);
```

Add a middleware to the route chain. Middlewares run in the order added, before the final handler. Must be called before `ecewo_route_handler()`.

### `ecewo_route_handler`

```c
void ecewo_route_handler(ecewo_route_t *route, ecewo_handler_t handler);
```

Finalize the route with its handler and register it with the router. After this call the builder must not be used again.

### `ecewo_route_register`

```c
void ecewo_route_register(ecewo_app_t *app, ecewo_method_t method, const char *path,
                          void **fns, int count);
```

Lower-level helper used internally by the `ECEWO_*` macros. `fns` is an array of `[middleware0, ..., middlewareN, handler]` and `count` is the total element count. Prefer the builder API in new code.

### Convenience macros

```c
ECEWO_GET    (app, path, ...);
ECEWO_POST   (app, path, ...);
ECEWO_PUT    (app, path, ...);
ECEWO_PATCH  (app, path, ...);
ECEWO_DELETE (app, path, ...);
ECEWO_HEAD   (app, path, ...);
ECEWO_OPTIONS(app, path, ...);
ECEWO_QUERY  (app, path, ...);
```

Variadic: one or more optional middlewares followed by exactly one handler.
`path` supports named parameters with a colon prefix (`"/users/:id"`) and a trailing wildcard (`"*"`). Routes are matched in registration order - first match wins.

```c
ECEWO_GET(app, "/users/:id", auth_middleware, get_user_handler);
```

See `02.defining-routes.md` for routing semantics.

## Per-request context

Lightweight key/value storage scoped to a single request, used to pass values from middleware to handler. Keys are compared by string; values are arbitrary `void *`.

### `ecewo_context_set`

```c
void ecewo_context_set(ecewo_request_t *req, const char *key, void *data);
```

Attach a value to the request under `key`. `data` must remain valid until the response is sent.

### `ecewo_context_get`

```c
void *ecewo_context_get(ecewo_request_t *req, const char *key);
```

Retrieve a value previously stored, or `NULL` if absent.

## Async task spawn

Run blocking work on the libuv worker thread pool. See `07.workers.md` for memory rules and full examples.

### `ecewo_spawn`

```c
int ecewo_spawn(ecewo_response_t *res,
                void *context,
                ecewo_spawn_handler_t work_fn,
                ecewo_spawn_done_t done_fn);
```

Run `work_fn(context)` on a worker thread, then call `done_fn(res, context)` on the event loop. Pass `res=NULL` for background tasks not tied to a request; pass the request `res` to offload blocking work inside a handler - `done_fn` receives the same `res` so `ecewo_send()` can be called there. Returns `0` on success, `-1` on error.

**Do not call any other `ecewo_*` function from inside `work_fn`** - it runs on a thread that does not own the event loop.

## Body streaming

See `09.body.md` for the full streaming model.

### `ecewo_body_stream`

```c
void ecewo_body_stream(ecewo_request_t *req, ecewo_response_t *res, ecewo_next_t next);
```

Middleware that switches the current request into streaming mode. Place it before your handler when you want to process the body incrementally. The handler runs before body data arrives; register data and end callbacks below to receive it. Without this middleware, the body is fully buffered and `ecewo_req_body()` is populated when the handler runs.

### `ecewo_body_on_data`

```c
void ecewo_body_on_data(ecewo_request_t *req, ecewo_body_data_cb_t callback);
```

Register a chunk callback.
- In streaming mode: called once per network chunk as data arrives.
- In buffered mode: called once synchronously with the complete body.

### `ecewo_body_on_end`

```c
void ecewo_body_on_end(ecewo_request_t *req, ecewo_response_t *res, ecewo_body_end_cb_t callback);
```

Register an end-of-body callback.
- In streaming mode: called after the last chunk is delivered.
- In buffered mode: called immediately if `ecewo_body_on_data()` has already been set.

### `ecewo_body_limit`

```c
size_t ecewo_body_limit(ecewo_request_t *req, size_t max_bytes);
```

Set the maximum allowed body size in bytes for this request. Default 10 MB. Requests exceeding the limit are rejected with `413 Payload Too Large`. Returns the previous limit. Must be called before body data starts arriving.

## App data and arena

For storing per-app state - useful for plugins and bindings.

### `ecewo_app_arena`

```c
ecewo_arena_t *ecewo_app_arena(const ecewo_app_t *app);
```

Return the app-lifetime arena. Allocations live until the app is shut down. Use in plugin init for config structs, connection pools, and other long-lived state.

### `ecewo_set_app_data`

```c
void ecewo_set_app_data(ecewo_app_t *app, void *key, void *data);
```

Store a pointer keyed by `key`. The recommended pattern is to use the address of a plugin-private file-static variable as the key - that guarantees uniqueness without coordination.

### `ecewo_get_app_data`

```c
void *ecewo_get_app_data(const ecewo_app_t *app, void *key);
```

Retrieve a pointer stored with `ecewo_set_app_data()`, or `NULL` if absent.

## Client refcounting

For plugin authors holding a client pointer across async boundaries.

### `ecewo_client_is_valid`

```c
bool ecewo_client_is_valid(ecewo_client_t *client);
```

Return `true` if the client's connection is still open.

### `ecewo_client_ref`

```c
void ecewo_client_ref(ecewo_client_t *client);
```

Increment the client's reference count, preventing it from being freed. Must be paired with a matching `ecewo_client_unref()`.

### `ecewo_client_unref`

```c
void ecewo_client_unref(ecewo_client_t *client);
```

Decrement the client's reference count. When it reaches zero the client is freed.

## Async work counter

A process-wide counter that keeps `ecewo_run()` blocking while non-request work is in flight (e.g. a plugin's background tasks).

### `ecewo_increment_async_work`

```c
void ecewo_increment_async_work(void);
```

Signal that an async operation has started. Must be balanced by `ecewo_decrement_async_work()`.

### `ecewo_decrement_async_work`

```c
void ecewo_decrement_async_work(void);
```

Signal that an async operation has completed.

## Event loop access

### `ecewo_get_loop`

```c
void *ecewo_get_loop(void);
```

Return the underlying `uv_loop_t *` (typed as `void *` so the public header does not require `uv.h`). Cast to `uv_loop_t *` to attach additional libuv handles. The loop is a process singleton - every app, timer, and `ecewo_spawn()` shares it.

## Connection takeover

For plugin authors implementing protocols on top of HTTP/1.1 (WebSocket, raw TCP after upgrade).

### `ecewo_takeover_config_new`

```c
ecewo_takeover_config_t *ecewo_takeover_config_new(void);
```

Allocate a new, zeroed takeover config. Returns `NULL` on allocation failure.

### `ecewo_takeover_config_free`

```c
void ecewo_takeover_config_free(ecewo_takeover_config_t *config);
```

Free a takeover config created by `ecewo_takeover_config_new()`.

### `ecewo_takeover_config_set_alloc_cb`

```c
void ecewo_takeover_config_set_alloc_cb(ecewo_takeover_config_t *config, void *alloc_cb);
```

Set the libuv `uv_alloc_cb` for the socket.

### `ecewo_takeover_config_set_read_cb`

```c
void ecewo_takeover_config_set_read_cb(ecewo_takeover_config_t *config, void *read_cb);
```

Set the libuv `uv_read_cb`. All inbound bytes are delivered to this callback after takeover.

### `ecewo_takeover_config_set_close_cb`

```c
void ecewo_takeover_config_set_close_cb(ecewo_takeover_config_t *config, void *close_cb);
```

Set the libuv `uv_close_cb`. Invoked when the connection is closed after takeover.

### `ecewo_takeover_config_set_user_data`

```c
void ecewo_takeover_config_set_user_data(ecewo_takeover_config_t *config, void *user_data);
```

Set the `user_data` pointer that will be available on the libuv handle's `data` field inside the alloc/read callbacks.

### `ecewo_connection_takeover`

```c
int ecewo_connection_takeover(ecewo_response_t *res, const ecewo_takeover_config_t *config);
```

Transfer ownership of the underlying TCP socket from ecewo to the caller. ecewo continues to track the connection (so idle cleanup and shutdown know about it) but stops reading from it. Typically called after sending an HTTP 101 upgrade reply. Returns `0` on success, `-1` on error.

### `ecewo_takeover_close_socket`

```c
void ecewo_takeover_close_socket(void *handle);
```

Close a taken-over socket cleanly. Triggers the `close_cb` registered with `ecewo_connection_takeover()`, then frees the underlying client. Plugins must use this rather than `uv_close()` directly so per-connection bookkeeping stays in sync.

### `ecewo_get_client_handle`

```c
void *ecewo_get_client_handle(ecewo_response_t *res);
```

Return the raw `uv_tcp_t *` for the connection (typed as `void *`). Cast to `uv_tcp_t *` to use with libuv APIs directly.

## Dynamic array and string builder macros

C-only ergonomic helpers built on top of the arena API. See `13.dynamic-array.md` and `14.string-builder.md` for full reference. They are not part of the FFI surface.

### `ECEWO_DA_APPEND(arena, da, item)`

Append `item` to a dynamic-array struct `da` (with `items` / `count` / `capacity` fields), growing through `ecewo_realloc()` as needed.

### `ECEWO_DA_APPEND_MANY(arena, da, new_items, new_items_count)`

Append a block of items at once.

### `ECEWO_SB_APPEND_CSTR(arena, sb, cstr)`

Append a null-terminated string's bytes (not including the NUL) to a string-builder dynamic array.

### `ECEWO_SB_APPEND_NULL(arena, sb)`

Append a single `0` byte to terminate a string built with `ECEWO_SB_APPEND_CSTR()`.

### `ECEWO__DA_INIT_CAP`

The initial capacity used by the dynamic-array macros (default `256`). Define it before including `ecewo.h` to override.

## Symbols not listed here

If a symbol exported from the library begins with `ecewo__` (double underscore), it is internal and is not part of the public API. It may change or disappear in any release. Public code must not use these symbols.
